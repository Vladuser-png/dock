3 спільні патерни

01

Пошук — головний вхід

Всі ставлять пошуковий рядок або фільтр-спеціальність у hero. Перший крок пацієнта — "знайти лікаря", а не "дізнатись про продукт".

02

Довіра через числа і верифікацію

Doc.ua: "6 900+ лікарів". Zocdoc: тільки відгуки з підтвердженим записом. Helsi: держ. реєстр. Кожен вирішує питання "чи можна довіряти" по-своєму.

03

Застосунок — основний канал

Всі мають App Store / Google Play на головній або в footer. Helsi: "в додатку зручніше". Продукт = застосунок, сайт = вхідна точка.

3 ключові відмінності

01

Ринок = монетизація

Zocdoc монетизує через лікарів, Doc.ua — через пацієнта/клініку, Helsi — через державу. Це повністю міняє UX-пріоритети.

02

Широта vs. глибина

Doc.ua бере широтою (аптека, лаби, хірургія). Zocdoc — глибиною (один flow, ідеально відточений). dock: вузько, але найчистіший UX.

03

Держава vs. комʼюніті vs. масштаб

Helsi довіряє через МОЗ. Zocdoc — через верифіковані відгуки. Doc.ua — через масштаб. dock будує довіру через UX-якість.

UX-патерни для "легко записатись до лікаря"

P1

Search-First

Велике поле пошуку в центрі. Аудиторія знає чого хоче. Ламається: каталог порожній.

P2

Guided Wizard

Питання → відповідь → результат. Підходить новачкам. Ламається: 5+ кроків = відтік. не для dock

P3

Availability-First

"Хто вільний сьогодні?" — показуємо слоти, потім лікаря. Як OpenTable. якщо є реальна база

P4

Map-First

Карта Києва з маркерами клінік. Підходить, якщо локація критична. Ламається на мобільному.

P5

App Showcase

Мокап застосунку в hero, скрін змінюється при скролі. Лендінг = маркетинг, а не продукт. рекомендовано

Вимір — Довіра до профілю лікаря · досвідчений дипломований спеціаліст

C1

Верифікація ліцензії

Чи перевіряє платформа ліцензію і показує це як видимий UI-сигнал (бейдж, підпис «перевірено»)

C2

Стаж роботи (числом)

Роки досвіду числом прямо на профілі — без кліку у деталі

C3

Академічний статус

Категорія (перша/вища), к.м.н./д.м.н., вчене звання, посада

C4

Рейтинг + к-ть відгуків

Чи показано скільки відгуків і як пояснений рейтинг — чи довіряти числу

C5

Gate відгуків

Відгук залишає тільки той, хто реально побував на прийомі (gate mechanism)

C6

Сертифікати та спеціалізація

Окремі сертифікати, курси, вузькі спеціальності — видно без копання у вкладки

C7

Лічильник прийомів

Соціальний доказ: «5 000 прийомів через платформу» — видимо на картці

C8

Trust на картці

Ключові сигнали довіри видні в каталозі без відкриття повного профілю

Топ-3 механізми для MVP
01

Стаж числом + академічний статус

«15 років · Вища категорія · к.м.н.» — три рядки, що зчитуються за 2 секунди. Клініка подає дані при підключенні, платформа не верифікує. Найдешевший trust-сигнал на MVP.

перенести
02

Значок «Перевірено платформою»

Бейдж верифікованої ліцензії через реєстр МОЗ (публічний API). Прибирає головний сумнів «а раптом він ненастпавжній лікар». Технічно: 1 day. Психологічно: ключовий gate для нового гравця.

перенести
03

Gate-механізм відгуків

Відгук = тільки якщо запис через dock + прийом підтверджено лікарем. Без цього — відгуки нічого не варті (кейс Helsi 2023). Статус «завершений прийом» → unlock форми відгуку.

перенести
Що не спрацює на MVP
✗ НЕ ПЕРЕНОСИТИ

AI-синтез відгуків (Helsi Plan)

Потребує ~500–1000 відгуків на лікаря, щоб рецензія мала зміст. На MVP перші місяці більшість лікарів матимуть 0–10 відгуків — AI видасть порожній або беззмістовний текст, що шкодить довірі.

Helsi запустив синтез до закриття проблеми фейкових записів: AI масштабує шум, а не сигнал. Без gate (C5) — AI-рецензія ненадійного = ненадійна вдвічі.

Інсайти для dock

Не копіюй агрегатор. Doc.ua вже є і він великий. dock виграє на якості UX і швидкості — менше кроків від відкриття до підтвердженого запису.
Лендінг ≠ застосунок. Конкуренти плутають ці два продукти. dock має зробити справжній маркетинговий лендінг, що продає ідею, а не функції.
Довіра — питання №1 для нового гравця. Варіант: фото реальних лікарів + прозорі відгуки + гарантія підтвердження.
Doctolib — найближчий аспіраційний еталон. Мінімалістичний дизайн, real photo, чистий пошук, акцент на швидкості — те саме для UA-ринку.
